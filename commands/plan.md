Before writing any code, analyze the task and propose a plan.

## Steps

1. **Interpret the ask.** If the task is ambiguous, ask clarifying questions before proceeding. If it's clear, skip this step.
2. **Explore what exists.** Read relevant files, check the current state. Don't assume.
3. **Identify the approach.** Outline what you'd change, create, or wire up. List the files involved.
4. **Flag risks or tradeoffs.** Anything that could break, any design decisions that have multiple valid paths.
5. **Estimate scope.** Roughly: is this a 5-minute tweak, a focused session, or a multi-step build?

## Rules

- Do NOT write or edit any code during planning.
- Keep the plan concise — bullet points, not paragraphs.
- Wait for my approval before executing.
